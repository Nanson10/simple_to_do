# simple_to_do

A lightweight Rust command-line to-do app that stores tasks in plain text files by day.

## Features

- Interactive terminal menu.
- Session default day for quick task entry.
- Add, edit, complete, and cancel tasks.
- Optional task due dates.
- Edit subcommands for text or due date updates.
- View unfinished tasks across days.
- Browse historical tasks by date.
- Pagination for long task lists.
- Plain-text storage (easy to inspect and back up).

## Requirements

- Rust toolchain (Cargo)

Install Rust if needed:

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

## Quick Start

From the project root:

```bash
cargo run
```

Or run release mode:

```bash
cargo run --release
```

Run with isolated test data directory (`test/` instead of `data/`):

```bash
cargo run -- --test
```

You can also use the helper script:

```bash
./run.sh
```

If needed, make it executable first:

```bash
chmod +x run.sh
```

## Main Menu

On startup, the app shows:

- Current session default day
- Unfinished task list (from today and earlier)
- Command options

Commands:

- `1` Add a task for the session default day
- `2` Edit a task (from to-do list or a specific day)
- `3` Complete a task from the to-do list
- `4` Cancel a task from the to-do list
- `5` View unfinished tasks (read-only)
- `6` Browse tasks by day
- `7` Set session default day
- `0` Exit

Edit subcommands (inside command `2`):

- `1` Edit task text
- `2` Edit task due date
- `0` Cancel

In paginated views:

- `1-7` Select item on current page
- `8` Previous page
- `9` Next page
- `0` Cancel/exit view

Tip: entering a non-number in command prompts displays help.

## Date Selection Rules

When prompted for a day, you can enter either:

- Day offset integer (relative to today)
  - `0` = today
  - `-1` = yesterday
  - `2` = two days from today
- Explicit date in `YYYY-MM-DD` format

## Data Storage

All app data is stored under `~/.simple_to_do/data/`.

- Daily files: `~/.simple_to_do/data/YYYY-MM-DD.txt`
- Generated unfinished summary: `~/.simple_to_do/data/to-do.txt`

Daily file format:

```text
# Tasks
# Format: [ ] pending, [x] completed, [~] cancelled; optional metadata tokens: (key:content)

[ ] Buy groceries
[ ] Buy groceries (due:2026-03-21)
[x] File taxes
[~] Reschedule dentist (note:called office)
```

Metadata token format:

- `(<key>:<content>)` appended to task text, separated by spaces.
- Current examples: `(due:2026-03-21)`, `(note:called office)`.
- Reserved characters in metadata content must be escaped with `\\`:
  - `:` becomes `\\:`
  - `(` becomes `\\(`
  - `)` becomes `\\)`
  - `\\` becomes `\\\\`

Status markers:

- `[ ]` pending
- `[x]` completed
- `[~]` cancelled

`~/.simple_to_do/data/to-do.txt` is auto-generated by the app. For consistency, update tasks through the CLI.

When using `--test`, the app writes files under `~/.simple_to_do/test/` and generates `~/.simple_to_do/test/to-do.txt`.

## Notes

- Unfinished list excludes future-dated tasks.
- Cancelling a task can append an optional note to task text.
- Pressing Enter on task edit deletes the selected task after confirmation.
- Pressing Enter when editing due date clears the due date.

## Precedence of tasks in to-do

Tasks in `to-do.txt` are ordered by:

- Due date precedence: tasks with due dates come first, and earlier due dates are higher priority (most overdue first).
- Task age fallback: if due dates are equal or missing, older task-day entries come first.
- Insertion order fallback: if still tied, earlier added tasks come first.
