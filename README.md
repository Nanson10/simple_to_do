# simple_to_do

A lightweight Rust command-line to-do app that stores tasks in plain text files by day.

## Features

- Interactive terminal menu.
- Session default day for quick task entry.
- Add, edit, complete, and cancel tasks.
- View unfinished tasks across days.
- Browse historical tasks by date.
- Pagination for long task lists.
- Plain-text storage (easy to inspect and back up).

## Requirements

- Rust toolchain (Cargo)

Install Rust if needed:

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

## Quick Start

From the project root:

```bash
cargo run
```

Or run release mode:

```bash
cargo run --release
```

You can also use the helper script:

```bash
./run.sh
```

If needed, make it executable first:

```bash
chmod +x run.sh
```

## Main Menu

On startup, the app shows:

- Current session default day
- Unfinished task list (from today and earlier)
- Command options

Commands:

- `1` Add a task for the session default day
- `2` Edit a task (from to-do list or a specific day)
- `3` Complete a task from the to-do list
- `4` Cancel a task from the to-do list
- `5` View unfinished tasks (read-only)
- `6` Browse tasks by day
- `7` Set session default day
- `0` Exit

In paginated views:

- `1-7` Select item on current page
- `8` Previous page
- `9` Next page
- `0` Cancel/exit view

Tip: entering a non-number in command prompts displays help.

## Date Selection Rules

When prompted for a day, you can enter either:

- Day offset integer (relative to today)
  - `0` = today
  - `-1` = yesterday
  - `2` = two days from today
- Explicit date in `YYYY-MM-DD` format

## Data Storage

All app data is stored under `data/`.

- Daily files: `data/YYYY-MM-DD.txt`
- Generated unfinished summary: `data/to-do.txt`

Daily file format:

```text
# Tasks
# Format: [ ] pending, [x] completed, [~] cancelled

[ ] Buy groceries
[x] File taxes
[~] Reschedule dentist (note: called office)
```

Status markers:

- `[ ]` pending
- `[x]` completed
- `[~]` cancelled

`data/to-do.txt` is auto-generated by the app. For consistency, update tasks through the CLI.

## Notes

- Unfinished list excludes future-dated tasks.
- Cancelling a task can append an optional note to task text.
- Pressing Enter on task edit deletes the selected task after confirmation.

## Future Plans

Potential next improvements for this project:

- Add recurring tasks (daily/weekly) with automatic carry-forward.
- Add task priorities (high/medium/low) and sort unfinished tasks by priority.
- Add due-time support in addition to date-only scheduling.
- Add tags and filtering (for example: `work`, `personal`, `errands`).
- Add search across all historical task files.
- Add an archive/export command (Markdown or CSV) for completed/cancelled tasks.
- Add undo for the latest action (complete/cancel/edit/delete).
- Add optional colorized terminal output for better readability.
- Add configurable data directory path through environment variable or config file.
- Add automated tests for storage parsing, date selection, and interactive flows.
